# Mobile UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Audit and improve the Titanium app mobile experience: capture screenshots at iPhone viewport, then implement responsive improvements so all pages are fully usable on a 390px screen.

**Architecture:** MembersTable gets a card-based layout below `md` breakpoint (the table stays for desktop). Toolbar filters move into a `Sheet` on mobile. Dashboard page and forms get targeted padding/spacing fixes. No new pages — only responsive improvements to existing components.

**Tech Stack:** Next.js 15, Tailwind CSS (mobile-first), shadcn/ui (Sheet, Badge, Button), Playwright (screenshots)

```ts
// scripts/mobile-screenshot.ts
import { chromium } from "@playwright/test";

async function capture() {
  const browser = await chromium.launch();
  const ctx = await browser.newContext({
    viewport: { width: 390, height: 844 },
    deviceScaleFactor: 2,
  });
  const page = await ctx.newPage();

  // Set BYPASS_AUTH cookie / header if needed — app uses BYPASS_AUTH env var server-side
  // Screenshots are taken against the local dev server (must be running)
  const base = "http://localhost:3000";

  await page.goto(`${base}/dashboard`);
  await page.waitForLoadState("networkidle");
  await page.screenshot({ path: "docs/mobile/screenshots-before/dashboard.png", fullPage: true });

  await page.goto(`${base}/dashboard/socios`);
  await page.waitForLoadState("networkidle");
  await page.screenshot({ path: "docs/mobile/screenshots-before/socios.png", fullPage: true });

  // Open create dialog
  const createBtn = page.getByRole("button", { name: /nuevo socio/i });
  if (await createBtn.isVisible()) {
    await createBtn.click();
    await page.waitForTimeout(400);
    await page.screenshot({ path: "docs/mobile/screenshots-before/create-dialog.png" });
  }

  await browser.close();
  console.log("Screenshots saved to docs/mobile/screenshots-before/");
}

capture().catch(console.error);
```

```tsx
                <div className="flex flex-col gap-0.5 shrink-0">

                    size="sm"

        {/* Desktop table view (hidden on mobile) */}
```

- [ ] **Step 2: Fix del grid de stats secundarias en el dashboard**

En `src/app/dashboard/page.tsx`, localizar:

```tsx
<div className="grid grid-cols-3 gap-3">
```

En 390px, 3 columnas con textos largos como "Vencen pronto" se recortarán. Cambiar a:

```tsx
<div className="grid grid-cols-3 gap-2 sm:gap-3">
```

Y en cada item, reducir el texto de la etiqueta en móvil. Localizar:

```tsx
<p className="text-[11px] text-muted-foreground uppercase tracking-wider font-medium truncate">
  Vencen pronto
</p>
```

El `truncate` ya ayuda. Pero adicionalmente, esconder la etiqueta en pantallas muy pequeñas y mostrar solo el icono + número:

```tsx
<p className="text-[10px] sm:text-[11px] text-muted-foreground uppercase tracking-wider font-medium truncate hidden xs:block sm:block">
  Vencen pronto
</p>
```

O más simple — dejar como está con `truncate` y verificar que no desborda.
